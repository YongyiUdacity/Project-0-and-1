Stage 1: Webpages, Documents, and Structure

Lesson 1: The Basics of the Web

How the Web Works

The main pieces in the web are:

1. Browser: a program that runs on user's computer to display files found on the web.

2. Internet: the world's largest computer network.

3. Servers: computers that host the files that make up the web.

The Browers makes request through Internet to Servers. These requests use HyperText Transfer Protocol (HTTP). The server responses to the requests with file that each Browser displays.

HTML

HTML stands for HyperText Markup Language. It is the main type of document on the web.

HTML is made up of: text content; markup; references to other documents; and links to other pages.

Tags and Elements

HTML markup is made up with tags. The tag is always contained in angled brackets.

An HTML element refers to everything within a set of opening and closing tags.

Attribute is a property of an HTML element.

Examples of tags listed in the lesson:

"b" is for bold.

"em" is for italic.

"a" is for linking another webpage with "href" attribute.

"img" is for linking an image with "src" attribute.

"br" is for adding a line space.

"p" is for creating a paragraph.

Inline Tags and Block Tags

Block tags are the tags created in an invisible box, such as "p" and "div".

Inline tags are just tags in the line, such as "a", "img", "br", and "span".

Lesson 2: Creating a Structured Document with HTML

The Tree Structure of HTML

Since the HTML elements can have other elements inside of them to form a hierarchical structure, You can draw this relationship like a family tree with nested boxes to represent each element.

How to Structure HTML

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them

Document Object Model (DOM) is a programming interface for HTML which can provide a structured representation of the document. More details about DOM can be found in DOM.

When you read an HTML documents as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

Lesson 3: Adding Style to Structure with CSS

Reasons to Avoid Repetition

1. Avoid repetition can reduce the minor errors when the programmer wants to make a change to repeated code.

2. Avoid repetition help keeping consistency of the style in a webpage.

CSS

CSS stands for Cascading Style Sheets. It is code written to control the "style" of related HTML elements.

One can use "class" attribute to apply style to the elements with the same class name.

How to include CSS Styling in a Web Page

1. Write CSS in the "head" of the HTML.

2. Link the HTML to a separate CSS file.

3. Write the style inline with the HTML.

Box Sizing and Positioning

HTML elements are boxes and each box has margin, border, padding and content.

To control the box size, you can set sizes in terms of percents rather than pixels, or you can set the "box-sizing" attribute to "border-box" for every element.

Divs are block elements which take up the entire width of a page. You can override this behavior by using "display: flex;" rule.
